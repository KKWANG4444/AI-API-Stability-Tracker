这份说明不列模型官方价格，也不做“中转一定比官方便宜”的承诺。模型价格、平台余额规则和支付门槛是三件不同的事，混在一起很容易误导。

## 先算总接入成本

评估一个 API 方案时，至少考虑：

- 模型调用费用；
- 支付和结算条件；
- 网络与部署成本；
- 多供应商客户端维护；
- 日志、重试、限流与告警；
- 故障迁移的工程时间。

个人测试和生产服务的权重不同。测试阶段可能更在意接入速度，生产阶段更在意可观测性、错误处理和合同条款。

## 不要复用过期价格表

模型和平台策略会调整。旧文章里的每百万 token 单价、折扣比例或“比官方便宜”结论，很快就会失效。

采购前应分别核验：

1. 模型厂商当前文档；
2. AI快站控制台的当前计费说明；
3. 账户所在地区的支付规则；
4. 目标工作负载的真实 token 使用量。

## 国内与国际支付不是一套规则

国内账户可用方式以控制台当前页面为准。

国际用户只能使用加密货币。**1 个 AI快站余额刀（“1刀”）= 0.07 USDC 或 0.07 USDT。** 国际用户不支持法币支付。付款前必须核对控制台支持的链和充值说明。

这里的“1刀”是平台余额单位，不是美元法币价格，也不是 USDC/USDT 的市场汇率。

## 用自己的请求量做预算

假设应用每天产生：

- 输入 token：`input_tokens_per_day`
- 输出 token：`output_tokens_per_day`
- 图片、音频或视频调用：`media_requests_per_day`

先从一周真实日志计算平均值和高峰值，再套用控制台当前计费规则。不要用一次演示请求估算整月费用。
